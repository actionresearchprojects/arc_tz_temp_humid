# ecovillage_t_h: Architectural Memory

Ecovillage temperature & humidity dashboard — static HTML build.
Combines TinyTag Excel loggers with Omnisense CSV sensors and Open-Meteo external data.

## Primary Files & Tools
- `build.py`: Regenerates `index.html` and `data/loggers.json`. Supports `--auto` mode using `sensor_snapshot.json` + fresh Open-Meteo/Omnisense.
- `fetch_openmeteo.py`: Fetches historical/forecast data (auto-fetched via GitHub Action).
- `fetch_omnisense.py`: Fetches Omnisense sensor data (auto-fetched via GitHub Action). Requires `OMNISENSE_USERNAME`/`OMNISENSE_PASSWORD` env vars.
- `index.html`: Main dashboard output (GitHub Pages). Fetches `data/config.json` at runtime via async `init()` to apply user overrides.
- `config.html`: Admin page for editing logger names/categories. Reads `data/loggers.json`, saves overrides to `data/config.json` via GitHub Contents API.
- `data/sensor_snapshot.json`: Pre-processed TinyTag/Govee sensor data (excludes auto-fetched sources).
- `data/omnisense/`: Auto-fetched Omnisense CSVs (tracked in git).
- `data/config.json`: User-edited overrides (tracked in git, edited via config.html). Fetched at runtime by index.html.
- `data/loggers.json`: Logger manifest with default names/categories (generated by build.py, read by config.html).
- `CLAUDE.md`: This file (Current project state/rules).
- `CHANGELOG.md`: Historical record of all changes (Read only when context is needed).

## Critical Technical Rules
- **Timezone**: Everything MUST use **East African Time (EAT, UTC+3)**. 
- **Timestamps**: When logging in `CHANGELOG.md`, use CST (UTC+8) per the user's local time: `### YYYY-MM-DD HH:MM:SS CST`.
- **Graphing**: Use `toEATString(ms)` helper in JS to prevent Plotly from converting to browser-local time.
- **Data Loading**: 
  - Historical Meteo: Used for adaptive comfort running mean.
  - Forecast Meteo: Shown as dashed lines; excluded from comfort calcs.

## Data Structure & Datasets
- **House 5**: 24 loggers (TinyTag + Omnisense + Open-Meteo). 
  - *Exclusions*: Omnisense `320E02D1`, `32760164` (outdoor/ceiling).
  - TinyTag `759498` (Bedroom 3 below metal roof): data filtered to before 2024-06-01 (moved to Schoolteacher's House on ~1 Jun 2024; Jun 1 excluded from both datasets).
- **Schoolteacher's House**: 3 TinyTag loggers. Uses TinyTag `861011` for adaptive comfort.

## UI Logic Constants
- **Threshold**: 32°C (red dotted line).
- **Adaptive Comfort**: Uses EN 15251 running mean (alpha=0.8). Default model: Vellei RH>60%.
- **Long-Term Mode**: Hides humidity, resets loggers to Open-Meteo, shows climate series.
- **Ordering**: Controlled by `sidebar_order` config; groups by room (TinyTag then Omnisense).

## Instructions for Claude
1. **Memory Hygiene**: Do NOT append logs to this file. 
2. **Logging**: Append all changes/updates to `CHANGELOG.md` with the CST timestamp.
3. **Builds**: After modifying `build.py` or `index.html`, remind the user to run `python build.py` to refresh the snapshot and dashboard.

## To update data
- **Automated**: GitHub Action `update-dashboard-data.yml` runs twice daily (04:00 & 16:00 UTC). Fetches both Open-Meteo and Omnisense data, then rebuilds with `--auto`.
- **Manual**: Only needed for TinyTag `.xlsx` data. Add files to `data/house5/` or `data/dauda/`, then run `python build.py`.
