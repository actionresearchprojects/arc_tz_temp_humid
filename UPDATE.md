# arc_tz_temp_humid — Data Update Guide

How to update the ecovillage temperature & humidity dashboard and push it to GitHub.

---

## Automated daily update (Open-Meteo)

A GitHub Action runs daily at 04:00 UTC (07:00 EAT) and automatically:
1. Fetches the latest historical and forecast data from Open-Meteo
2. Rebuilds `index.html` using the sensor snapshot + fresh Open-Meteo data
3. Commits and pushes if anything changed

**No user intervention needed** — the dashboard's external temperature data stays current automatically.

To trigger it manually: go to the repo's **Actions** tab → **Update Open-Meteo data** → **Run workflow**.

### First-time setup

In the repo's **Settings → Actions → General → Workflow permissions**, ensure **Read and write permissions** is selected. Without this, the Action cannot push commits.

---

## Manual update: sensor data (TinyTag / Omnisense)

Only needed when you have new sensor data files to add.

### Step 1 — Open Terminal and navigate to the project folder

**Mac (quickest way):**
Open Finder, navigate to the `arc_tz_temp_humid` folder, then right-click it → **New Terminal at Folder**

**Or open Terminal manually:**
- Open Terminal (Applications → Utilities → Terminal, or Spotlight: type `Terminal`)
- Then type:
  ```
  cd ~/Downloads/ARC_graphs/arc_tz_temp_humid
  ```
  Press Enter. If you saved the folder somewhere else, adjust the path accordingly.

Confirm you're in the right folder:
```
pwd
```
Should show something like `/Users/yourname/Downloads/ARC_graphs/arc_tz_temp_humid`

---

### Step 2 — Update data files

- **TinyTag .xlsx files** — add or replace files in `data/house5/` and/or `data/dauda/`
- **Omnisense CSV** — see `OMNISENSE_DATA_UPDATE_GUIDE.md` (one level up in `ARC_graphs/`) for download instructions

---

### Step 3 — Rebuild the dashboard (full build)

```
python build.py
```

This reads all sensor data files + Open-Meteo data and regenerates `index.html`. It also saves `data/sensor_snapshot.json` — this file is used by the daily automated update so it doesn't need the raw sensor files.

---

### Step 4 — Push to GitHub

```
git add index.html data/sensor_snapshot.json data/openmeteo/ data/hist_proj/
git commit -m "update sensor data"
git push
```

Or as a one-liner:
```
git add index.html data/sensor_snapshot.json data/openmeteo/ data/hist_proj/ && git commit -m "update sensor data" && git push
```

To push code changes (e.g. edits to `build.py`, `CLAUDE.md`, `UPDATE.md`):
```
git add build.py CLAUDE.md UPDATE.md && git commit -m "describe your change" && git push
```

If this is the first time pushing from this machine, see **GitHub authentication** below.

---

### Step 5 — Verify on GitHub Pages

The live dashboard is at:
**https://actionresearchprojects.github.io/arc_tz_temp_humid/**

GitHub Pages refreshes within a minute or two of pushing. Hard-refresh the browser (Cmd+Shift+R on Mac) if you're not seeing the latest version.

## GitHub authentication (first time only)

GitHub no longer accepts your account password over HTTPS. You need a **Personal Access Token (PAT)**:

1. Go to [github.com](https://github.com) → your profile → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
2. Click **Generate new token (classic)**
3. Give it a name, set an expiry, and tick the **repo** scope
4. Copy the token — use it as your "password" when Git asks

Alternatively, use the GitHub CLI (simpler):
```
gh auth login
```
Follow the prompts — this handles authentication for all future pushes automatically.

## What gets pushed to GitHub

Files tracked in the repo:
- `index.html` — the dashboard (regenerated by `build.py`)
- `build.py` — the build script
- `fetch_openmeteo.py` — Open-Meteo API fetcher (used by GitHub Action)
- `data/openmeteo/` — latest Open-Meteo CSVs (historical + forecast)
- `data/sensor_snapshot.json` — pre-processed sensor data for automated builds
- `data/hist_proj/` — Copernicus ERA5 + SSP climate projection data
- `CLAUDE.md` — project notes
- `UPDATE.md` — this file
- `logo.png`
- `.gitignore`
- `.github/workflows/update-openmeteo.yml` — daily automation

Private data that stays local (gitignored):
- `data/house5/` — TinyTag .xlsx files
- `data/dauda/` — TinyTag .xlsx files
- `data/omnisense_*.csv` — Omnisense CSV
- `data/open-meteo*.csv` — legacy single Open-Meteo CSV (if still present)
- `data/openmeteo/legacy/` — archived previous Open-Meteo fetches

GitHub repo: **https://github.com/actionresearchprojects/arc_tz_temp_humid**
