# arc_tz_temp_humid — Data Update Guide

How to update the ecovillage temperature & humidity dashboard and push it to GitHub.

---

## Automated update (Open-Meteo + Omnisense)

A GitHub Action runs **twice daily** at 04:00 and 16:00 UTC and automatically:
1. Fetches the latest historical and forecast data from Open-Meteo
2. Fetches the latest Omnisense sensor data (last 90 days)
3. Rebuilds `index.html` using the sensor snapshot + fresh Open-Meteo + Omnisense data
4. Commits and pushes if anything changed

**No user intervention needed** — external temperature and Omnisense sensor data stay current automatically.

To trigger it manually: go to the repo's **Actions** tab → **Update dashboard data** → **Run workflow**.

### First-time setup

1. In the repo's **Settings → Actions → General → Workflow permissions**, ensure **Read and write permissions** is selected. Without this, the Action cannot push commits.

2. Add Omnisense credentials as GitHub Secrets:
   - Go to **Settings → Secrets and variables → Actions → New repository secret**
   - Add `OMNISENSE_USERNAME` with your Omnisense login username
   - Add `OMNISENSE_PASSWORD` with your Omnisense login password

## Manual update: TinyTag sensor data

Only needed when you have new TinyTag `.xlsx` files to add.

### Step 1 — Open Terminal and navigate to the project folder

**Mac (quickest way):**
Open Finder, navigate to the `arc_tz_temp_humid` folder, then right-click it → **New Terminal at Folder**

**Or open Terminal manually:**
- Open Terminal (Applications → Utilities → Terminal, or Spotlight: type `Terminal`)
- Then type:
  ```
  cd ~/Downloads/ARC_graphs/arc_tz_temp_humid
  ```
  Press Enter. If you saved the folder somewhere else, adjust the path accordingly.

Confirm you're in the right folder:
```
pwd
```
Should show something like `/Users/yourname/Downloads/ARC_graphs/arc_tz_temp_humid`

---

### Step 2 — Update data files

- **TinyTag .xlsx files** — add or replace files in `data/house5/` and/or `data/dauda/`

---

### Step 3 — Rebuild the dashboard (full build)

```
python build.py
```

This reads all sensor data files + Open-Meteo + Omnisense data and regenerates `index.html`. It also saves `data/sensor_snapshot.json` — this file contains only TinyTag/Govee data and is used by the automated update so it doesn't need the raw `.xlsx` files.

---

### Step 4 — Push to GitHub

```
git add index.html data/sensor_snapshot.json data/openmeteo/ data/omnisense/ data/hist_proj/ data/loggers.json
git commit -m "update sensor data"
git push
```

Or as a one-liner:
```
git add index.html data/sensor_snapshot.json data/openmeteo/ data/omnisense/ data/hist_proj/ data/loggers.json && git commit -m "update sensor data" && git push
```

To push code changes (e.g. edits to `build.py`, `CLAUDE.md`, `UPDATE.md`):
```
git add build.py CLAUDE.md UPDATE.md && git commit -m "describe your change" && git push
```

If this is the first time pushing from this machine, see **GitHub authentication** below.

---

### Step 5 — Verify on GitHub Pages

The live dashboard is at:
**https://actionresearchprojects.github.io/arc_tz_temp_humid/**

GitHub Pages refreshes within a minute or two of pushing. Hard-refresh the browser (Cmd+Shift+R on Mac) if you're not seeing the latest version.

## GitHub authentication (first time only)

GitHub no longer accepts your account password over HTTPS. You need a **Personal Access Token (PAT)**:

1. Go to [github.com](https://github.com) → your profile → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
2. Click **Generate new token (classic)**
3. Give it a name, set an expiry, and tick the **repo** scope
4. Copy the token — use it as your "password" when Git asks

Alternatively, use the GitHub CLI (simpler):
```
gh auth login
```
Follow the prompts — this handles authentication for all future pushes automatically.

## What gets pushed to GitHub

Files tracked in the repo:
- `index.html` — the dashboard (regenerated by `build.py`)
- `config.html` — admin page for editing logger names and categories (see below)
- `build.py` — the build script
- `fetch_openmeteo.py` — Open-Meteo API fetcher (used by GitHub Action)
- `fetch_omnisense.py` — Omnisense data fetcher (used by GitHub Action)
- `data/openmeteo/` — latest Open-Meteo CSVs (historical + forecast)
- `data/omnisense/` — latest Omnisense CSV (auto-fetched)
- `data/sensor_snapshot.json` — pre-processed TinyTag/Govee data for automated builds
- `data/hist_proj/` — Copernicus ERA5 + SSP climate projection data
- `data/config.json` — user-edited logger name/category overrides (edited via config.html)
- `data/loggers.json` — logger manifest generated by `build.py` (read by config.html)
- `CLAUDE.md` — project notes
- `UPDATE.md` — this file
- `logo.png`
- `.gitignore`
- `.github/workflows/update-dashboard-data.yml` — twice-daily automation

### Config admin page

**`config.html`** is a separate GitHub Pages page that lets you:
- Rename any logger (changes the display label on all charts)
- Change a logger's category between Room / Structural / Other (affects sidebar grouping and adaptive comfort eligibility)

To use it:
1. Go to `https://actionresearchprojects.github.io/arc_tz_temp_humid/config.html`
2. Generate a GitHub PAT with **Contents: Read & Write** scope on this repo (link on the page)
3. Enter the PAT and click **Load config**
4. Edit names / categories in the table
5. Click **Save changes**

Changes take effect immediately on the next dashboard load (no build required — the dashboard fetches `data/config.json` at runtime).

Private data that stays local (gitignored):
- `data/house5/` — TinyTag .xlsx files
- `data/dauda/` — TinyTag .xlsx files
- `data/omnisense_*.csv` — legacy Omnisense CSV in data/ root (if still present)
- `data/omnisense/legacy/` — archived previous Omnisense fetches
- `data/open-meteo*.csv` — legacy single Open-Meteo CSV (if still present)
- `data/openmeteo/legacy/` — archived previous Open-Meteo fetches
